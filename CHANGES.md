# Финальная сборка v2.3.0 — Русский Урожай

**Дата:** 2026-05-06

## Что нового в v2.3.0

### Основные фичи
- **Чат-модал** с историей сообщений, Realtime, кнопками переходов статуса сделки.
- **State machine сделок:** `pending → paid → shipping → delivered → completed`.
- **Анонимность:** контрагенты видят только handle (`B-1284` / `S-7821`), без `company_name` / ИНН / телефона.
- **Кнопки «Откликнуться» и «Сделать ценовое предложение»** теперь открывают чат.
- **Админ-панель:**
  - Новый модал «💬 Чаты платформы» — обзор всех переговоров с фильтрами.
  - В таблице «Все сделки» — кнопка чата для каждой сделки.
  - Версия и changelog в правом нижнем углу панели — клик открывает журнал релизов.

### Динамические счётчики (фикс)
- Чипсы «Пшеница 8», «Ячмень 4» и т.д. на `/catalog.html` теперь **пересчитываются** из БД после загрузки данных (раньше были статичные).
- Счётчик «Активные» на `/sale.html` тоже пересчитывается.

### Версионирование
- В `assets/js/config.js` есть `VERSION` и `CHANGELOG`.
- В админ-панели внизу — плашка с версией. Клик открывает модал с историей релизов.
- При каждом обновлении нужно: bump `VERSION`, добавить запись в `CHANGELOG` (наверх).

### Серверные миграции (накатано в прод)
| Миграция | Что делает |
|---|---|
| `privacy_anon_profiles_view` | View `profiles_public` (handle вместо PII) + блокировка чужих профилей через RLS |
| `chat_and_deals_rpc` | RPC: respond_to_offer/request, advance_deal (state machine), create_deal_from_thread, sendMessage |
| `enable_realtime_chat` | Realtime включён на `messages`, `message_threads`, `deals` |
| `fix_profiles_public_security_definer` | Фикс: view не должна была быть с security_invoker |
| `admin_threads_visibility` | Админам — доступ ко всем тредам и сообщениям. RPC: `admin_list_threads`, `admin_post_message` |

### Критичные фиксы
- `if` без скобок в `handleProposal` / `handleRespond` — обе кнопки молча умирали.
- `data-offer-id="demo"` на product.html заменяется на реальный UUID при загрузке страницы.
- Утечки `company_name` в `renderDealRow` и `renderRequestCard` — везде заменено на handle.
- View `profiles_public` пересоздана без `security_invoker` — это ломало JOIN с продавцами для не-админов.

## Сценарий тестирования

### Покупатель ↔ Продавец
1. С аккаунта продавца открыть `/sale.html` → «Откликнуться» на заявке → ввести цену/объём/сообщение → откроется чат.
2. С аккаунта покупателя зайти в кабинет → блок «Активные переговоры» → новый чат с handle продавца.
3. Покупатель в чате: «🤝 Оформить сделку» → ввести цену + объём → создаётся сделка `pending`.
4. Покупатель: «Я оплатил» → `paid`. Системное сообщение в чате.
5. Продавец: «Отгрузил товар» → `shipping`. Затем «Доставлено» → `delivered`.
6. Покупатель: «Подтверждаю получение» → `completed`. Эскроу разблокирован.

### Админ — контроль
1. Зайти под админом → в админ-панели:
   - **«💬 Чаты платформы»** — список всех чатов. Открой любой → видно переписку, можно написать как 👮.
   - **«Все сделки»** — таблица. У каждой строки 💬-кнопка для открытия чата + дропдаун статуса.
   - **«Споры и жалобы»** — отфильтрованные сделки в `disputed`.
   - Клик по плашке версии внизу — журнал релизов.

### Каталог и заявки
1. `/catalog.html` — счётчики «Пшеница N», «Ячмень N» и т.д. соответствуют реальной БД.
2. Когда добавляешь новый оффер через админку, после деплоя/рефреша чипсы должны обновиться.

## Как добавлять следующие версии

В `site/assets/js/config.js`:

```js
VERSION: '2.4.0',                  // ← bump
BUILD_DATE: '2026-05-XX',          // ← обнови

CHANGELOG: [
  {
    version: '2.4.0',              // ← новая запись наверх
    date: '2026-05-XX',
    summary: 'Краткое описание релиза',
    changes: [
      'Что добавлено',
      'Что исправлено',
      // ...
    ]
  },
  {
    version: '2.3.0',              // ← старые ниже, не удаляй
    ...
  },
]
```

После пуша админ увидит новую запись в журнале релизов.
