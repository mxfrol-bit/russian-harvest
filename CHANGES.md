# Финальная сборка v2.6.15 — Русский Урожай

**Дата:** 2026-05-12

## Главное в v2.6.15

Точечный хотфикс — **не разворачивались подкатегории культур в каталоге**.

### Симптом

На мобильной версии (ширина <1100px) `/catalog.html` → фильтр «Культура» → стрелки ▾ рядом с Пшеницей, Ячменём, Кукурузой, Тритикале, Горохом, Соей, Нутом и Чечевицей не разворачивали подкатегории при клике.

### Корневая причина

Гибридная логика на трёх уровнях:

1. HTML: `<div class="filter-subitems" ... hidden>`
2. Браузерный дефолт: `[hidden]{display:none}` (специфичность 10)
3. Кастомный CSS: `.filter-subitems[hidden]{display:flex !important;}` (специфичность 20 + `!important`)
4. Класс-переключатель: `.filter-subitems.expanded{max-height:400px;opacity:1}`

Расчёт говорил, что `!important` должен победить дефолт. **На практике** — в некоторых рендер-цепочках (особенно когда родительский `.filters-aside` сам имел `display:none` на мобиле, потом получал `.open` и становился `display:block`) браузер всё равно применял `display:none` от `[hidden]`. Без `display:flex` блок не существовал в флоу — `max-height` транзишн не запускался.

### Что починено

| Слой | Было | Стало |
|---|---|---|
| HTML | `<div class="filter-subitems" data-subitems="wheat" hidden>` | `<div class="filter-subitems" data-subitems="wheat">` (12 блоков очищены) |
| CSS | Костыль `.filter-subitems[hidden]{display:flex !important}` | Правило удалено — видимость через класс `.expanded` |
| JS (страховка) | — | `document.querySelectorAll('.filter-subitems[hidden]').forEach(s => s.removeAttribute('hidden'))` на инициализации |
| build.py | Генерировал `hidden` атрибут | Атрибут убран из шаблона — не вернётся при ребилде |
| Click-handler | Просто `addEventListener` | + `e.preventDefault()` + `e.stopPropagation()` (защита от всплытия к label) |
| Размер кнопки | 26×26 px | 32×32 px (лучшая попадаемость на тач) |
| Touch-оптимизация | — | `touch-action:manipulation` (нет 300ms задержки iOS) + `-webkit-tap-highlight-color:transparent` |
| z-index | — | `position:relative; z-index:2` (гарантия что кнопка над label-слоем) |

### Файлы изменены

- `site/assets/js/main.js` — обработчик `.crop-toggle` + строка снятия `hidden` на инициализации
- `site/assets/css/main.css` — упрощены `.crop-toggle` и `.filter-subitems`, удалён костыль `[hidden]`
- `site/catalog.html` — удалены 12 атрибутов `hidden` из `.filter-subitems`
- `scripts/build.py` — `hidden` убран из шаблона генерации
- `site/assets/js/config.js` — VERSION → 2.6.15

### 📐 Расчёт расстояний через БД

Раньше на карточках было фейковое «18 км до вас» из seed-данных. Теперь:

- В `geo_units` у каждого города есть `lat`/`lng`
- На таблицах `offers` и `buyer_requests` появилось поле `city_id` (FK на `geo_units`)
- SQL-функция `geo_distance_km(lat1,lng1,lat2,lng2)` — Haversine, IMMUTABLE, можно кешировать
- RPC `offers_with_distance(p_lat, p_lng)` — отдаёт офферы с уже посчитанным `distance_km`, отсортированные:
  ```
  is_premium DESC NULLS LAST,        -- VIP сверху
  distance_km ASC,                   -- ближайшие
  created_at DESC                    -- свежие
  ```
- В `api.js` метод `listOffersWithDistance()` автоматически берёт координаты залогиненного пользователя из `profile.city_lat/city_lng`, или НН (`56.3269, 44.0075`) для гостей
- `syncCatalog` подменяет лейбл «Регион → Нижний Новгород» на реальный `cityFrom → user_city`

**Smoke-test:** НН → Ворсма = **59.5 км** ✓

## Известные ограничения

1. **Привязка `city_id` к существующим офферам** — старые офферы в БД (которые ты создал до v2.5.1) не имеют `city_id`. Расстояние для них не посчитается, в карточке будет `—`. **Решение:** при создании нового оффера через UI — добавить выбор города из каскада (Регион → Район → Город) и сохранять `city_id`. Сейчас в форме «Создать оффер» этот каскад **не подключен** (только в «Создать заявку»). Если хочешь — добавлю в v2.5.2.

2. **Уровень 2 для не-Нижегородских регионов** — для других регионов сейчас прямой маппинг регион → город (без района). Это упрощённая схема, для прода её можно расширить через INSERT в `geo_units` без миграции.

3. **Старые seed-аккаунты** — в Supabase Auth остался один `admin@russian-harvest.ru` (с ролью `buyer`, не admin — никто не получил привилегий). Можно удалить вручную через Supabase Dashboard если надо.
